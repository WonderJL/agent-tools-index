# CLI pillar

The CLI pillar is the system's collection of agent-facing command-line tools — 50 items, all authored in-house (none vendored). Each one wraps a backend, API, or workflow behind a single executable so that any coding agent can drive it through ordinary shell execution rather than a bespoke integration. They are the hands-on capabilities of the monorepo: where skills carry know-how and sub-agents carry personas, CLIs do the actual work against real services.

Every tool is built with the CLI-Anything methodology and the **AXI (Agent eXperience Interface)** output contract, which is what makes them legible to an agent reading non-TTY stdout. Structured output is **TOON only** — there is no `--json` mode; a shared serializer in the `_shared` package (`cli_anything._shared.toon`) renders all output, so individual CLIs never hand-roll a format. The contract is deliberately terse: default list schemas carry 3–4 fields, list output ships pre-computed aggregates and a `count: N of M total`, detail views truncate large fields and report the full size, and empty results state the zero case explicitly and exit 0. Tools follow uniform conventions — `cli_anything.*` namespace packaging, a `cli-<name>` console-script entry point, Python ≥3.10, and `CLI_ANYTHING_<NAME>_*` environment variables for config — and are tested against real backends before they ship.

The fleet clusters into a few families. The largest is **AI / agent orchestration** (15 tools), spanning delegation, parallel-agent coordination, and session buses; notable examples include `cli-ai-crew` (a harness-agnostic bus that pushes prompts into labelled Claude/Codex worker sessions and tails their transcripts), `cli-agent-parallel-manager` (parallel local agents via per-task git worktrees and on-disk locks), and `cli-ai-usage` (a cron-safe usage tracker across Codex, Claude, Cursor, and Gemini). Other families cover **diagram / media / render** (8), **skill / agent infra** (7) — including the foundational `cli-_shared` primitives — **git / dev workflow** (5), **doc / OCR / conversion** (5), and **productivity / SaaS** integrations (4).
