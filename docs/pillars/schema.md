# Schema pillar

The schema pillar is the system's set of JSON Schema contracts. Where other pillars hold capabilities (skills, agents, CLIs) or composition logic (kits, hosts), this pillar holds the validation rules that keep the structured artifacts those pillars depend on well-formed and reproducible. If a manifest, catalog, or activity record is malformed, the schema is what catches it before a renderer, gate, or downstream tool ever reads it.

All 6 contracts are authored (none vendored) and written against JSON Schema draft 2020-12. Each declares a `title`, a `description`, and a strict shape — most set `additionalProperties: false` and pin a constant `apiVersion`/`schema_version` so a document either matches exactly or fails closed, never half-validating. They fall into two groups. The first validates the build inputs of the larger system: `integration.yaml` defines the integration-package manifest (pinned sources with SHA-256, install steps, file-placement strategies, env-var contracts, lifecycle hooks, and verification checks for per-host setup), and the **agent-tools-index catalog** schema constrains the public catalog itself, requiring every entry to declare its pillar, id, name, category, description, and `origin` (authored vs vendored). The second group is the **Work Receipts** family — `session-receipt`, `work-item-report`, `public-export`, and `export-review` — a chain of records that capture an AI coding session's timing, token and cost usage, complexity and quality bands, and, critically, a privacy classification and publish-safety recommendation that gates whether a scrubbed public export is allowed.

This pillar is the enforcement edge of two principles the system leans on heavily: *one source of truth, deterministic renderers*, and *allowlist at trust boundaries*. The catalog schema's required `origin` field is exactly what keeps authored and vendored work distinguishable, and the public-export schema's privacy guards are part of the same fail-closed posture that protects the private-to-public boundary described in the [Architecture](../architecture.md).
